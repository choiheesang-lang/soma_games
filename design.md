# 소마챔피언즈 미니게임 디자인 가이드 (`design.md`)

> 100개의 미니게임이 **"하나의 제품"으로 보이게** 만드는 최소한의 공통 규칙입니다.
> 버전 v0.3 · 2026.06.10 · 작성: 디자인 · 대상: 기획자 / 개발자

---

## 문서 구성 (총 3개)

GUI(캐릭터·로고·배경)를 만드는 방식이 **AI 생성**과 **소마 브랜드**로 갈리기 때문에, 가이드를 3개로 나눴습니다.

| 문서 | 내용 | 누가 읽나 |
|---|---|---|
| **`design.md`** (이 문서) | 모든 게임 공통 규칙 — 토큰, 컴포넌트, 화면 흐름, SVG, 게임 플레이 자유도 | **전원 필수** |
| `design_AI.md` | GUI를 **AI로 생성**할 때의 추가 규칙 | AI 생성 게임 담당 |
| `design_soma.md` | GUI를 **소마 브랜드 자산**으로 만들 때의 추가 규칙 | 소마 캐릭터 사용 게임 담당 |

> **순서** — 먼저 이 `design.md`를 읽고, 게임의 GUI 방식에 따라 `design_AI.md` **또는** `design_soma.md` 중 **하나만** 추가로 읽으세요. 두 방식은 **혼합하지 않습니다** (6번 참고).

## 이 문서를 읽는 법

- **기획자** — 게임을 정의할 때 "무엇이 고정이고 무엇이 자유인지"를 확인하세요. 콘셉트·캐릭터는 사업부와 협의하되, 컬러·폰트·컴포넌트는 여기 규칙을 따릅니다.
- **개발자** — 구현 시 이 문서 + `tokens.css` + Figma 컴포넌트 라이브러리를 기준으로 작성하세요. **AI 코딩 도구를 쓴다면 각 게임 저장소에 이 `design.md`를 컨텍스트로 포함**시키고, 토큰과 에셋은 직접 import / export 하세요.

핵심은 **두 가지 원칙**(이모지 금지·자유도 3단계)입니다. 이것만 이해하면 나머지는 참조용입니다.

## 0. 핵심 원칙

### 0-1. 이모지 금지 → SVG 사용 (필수)

미니게임의 모든 아이콘·기호·장식 요소는 **이모지(🎮🔊✅)가 아니라 SVG**로 만듭니다.

- **해상도** — 아이패드부터 전자칠판까지, 다양한 화면에서 그래픽이 깨지지 않습니다. 벡터인 SVG만 이걸 

### 0-2. 자유도 3단계

100개 게임의 "재미"는 살리되 "통일감"은 잃지 않기 위한 핵심 장치입니다.

| 영역 | 자유도 | 포함 | 규칙 |
|---|---|---|---|
| **UI 시스템** | **0%** | 공통 컴포넌트(스플래시·버튼·인풋/셀렉트·게임소개·게임팝업·체크/라디오), 디자인 토큰(폰트·간격·라운드) | 100개 게임 전부 동일 |
| **GUI 아이덴티티** | **0~50% (방식별)** | 캐릭터, 게임 로고·타이틀, 게임 배경 | 방식에 따라 `design_AI.md` / `design_soma.md` 참고 |
| **게임 플레이** | **100%** | 인게임 상호작용, 이펙트, 애니메이션 타이밍 | 완전히 자유 (브랜드 컬러 사용 권장) |

요약: **틀(UI)은 고정, 아이덴티티는 방식별 부분 자유, 게임 플레이는 완전 자유.**

## 1. 공통 화면 흐름

모든 게임이 거치는 흐름입니다. **스플래시·소개·완료 화면의 컴포넌트는 공통 규칙(4번)을 따르고**, 인게임은 자유(7번)입니다.

```
스플래시(진입·로딩) → 게임 소개(시작) → 게임 선택 → 게임 진행(플레이) → 보상 → 완료(재시작·다음·홈)
```

| 단계 | 화면 | 비고 |
|---|---|---|
| 진입 | 스플래시스크린 | 로딩. 게임 로고가 화면 중앙에 표시 |
| 소개 | 게임 소개 화면 | 게임 타이틀·설명, 시작 버튼 |
| 선택 | 게임 선택 | 난이도·모드 등. **게임에 따라 생략 가능** |
| 플레이 | 인게임 | 화면·상호작용 자유 (7번) |
| 보상 | 보상 연출 | **게임별 고유 연출** (7번) |
| 완료 | 결과창 | 인포그래픽, 다시하기·다음·홈 |

## 2. 디자인 토큰 (자유도 0% — 그대로 사용)

값은 모두 `tokens.css`에 정의돼 있습니다. 아래는 참조용 요약이며, 코드에서는 `var(--token)`으로 사용하세요.

### 2-1. 컬러

**Primary (브랜드 컬러)**

| 토큰 | 값 | 용도 |
|---|---|---|
| `--color-primary` | `#F19A2F` | 주황 — 핵심 강조 |
| `--color-primary-green` | `#69BE00` | 녹색 — 보조 강조 |
| `--color-ink` | `#222824` | 브랜드 검정 |

> 게임별로 컬러를 **1~2개(최대 3개)** 추가할 수 있습니다. 여러 색을 흩뿌리지 말고, 추가한 색의 **명도 단계를 나눠** 쓰세요.

**텍스트 (검정 + 투명도)** — 밝은 배경 기준. 어두운 배경에서는 흰색에 같은 단계 투명도(`--text-on-dark-*`).

| 토큰 | 투명도 | 용도 |
|---|---|---|
| `--text-1` | 88% | 최강조 제목 |
| `--text-2` | 78% | 본문 강조 |
| `--text-3` | 58% | 본문 |
| `--text-4` | 38% | 보조 |
| `--text-5` | 30% | 비활성 |
| `--text-6` | 18% | 플레이스홀더 |

**라인 / 보더 (검정 + 투명도)** — 어두운 배경에서는 흰색 버전 사용.

| 토큰 | 투명도 |
|---|---|
| `--line-1` | 55% |
| `--line-2` | 37% |
| `--line-3` | 13% |
| `--line-4` | 9% |

**상태값 (Semantic)** — 의미 전달용. 브랜드 주황(`#F19A2F`)과 경고 주황(`#E19F3C`)은 **역할이 다르니** 섞지 마세요.

| 토큰 | 값 | 의미 |
|---|---|---|
| `--status-danger` | `#D23040` | 위험 / 오답 |
| `--status-warning` | `#E19F3C` | 경고 |
| `--status-success` | `#4DB661` | 성공 / 정답 |
| `--status-accent` | `#50B4BB` | 강조 포인트 |
| `--status-info` | `#4571C6` | 정보 |
| `--status-neutral` | `#6F6F6F` | 중립 |

### 2-2. 타이포그래피

- **브랜드 폰트** — **CookieRun** (기본). 모든 게임의 기본 폰트입니다.
- **게임 스타일별 폰트** — 게임 성격에 맞게 아래에서 **자유롭게 선택**해 사용하세요.

| 게임 스타일 | 폰트 | 토큰 |
|---|---|---|
| 기본 / 브랜드 | CookieRun | `--font-brand` |
| 레트로 | DNF Bit Bit v2, NeoDunggeunmo Pro | `--font-game-retro` |
| 공·스포츠 | NEXON Football Gothic | `--font-game-sports` |
| 레이싱·자동차·속도 | Kartrider | `--font-game-racing` |
| 스토리·귀여움 | Maplestory | `--font-game-story` |
| 전쟁·공포 | Warhaven | `--font-game-war` |

- **폰트 개수 규칙** — 한 게임에 폰트는 가능하면 **1종**, 최대 2종. **2종 이상 쓸 경우 반드시 브랜드 폰트(CookieRun)를 포함**합니다. (tokens.css의 각 스타일 폰트는 CookieRun을 폴백으로 두어 이 규칙이 자동 충족됩니다.)
- **사이즈 스케일** — `20 / 24 / 36 / 40 / 46 / 52 / 64 px` (`--font-size-1` ~ `--font-size-7`)
- **텍스트 컬러** — 가독성이 중요한 텍스트는 텍스트 토큰(검정+투명도, 특히 38·58·78%)을, 강조에만 부분적으로 브랜드/포인트 컬러를 씁니다.

### 2-3. 간격 · 라운드 (8의 배수)

- **간격(spacing)** — `8 / 16 / 24 / 32 / 40 / 48` (모두 8의 배수, `--space-1` ~ `--space-6`)
- **라운드(radius)** — 동일하게 8의 배수 (`8 / 16 / 24`, 완전한 원은 `--radius-full`)

### 2-4. tokens.css

위 값은 모두 `tokens.css`에 변수로 정의돼 있습니다. 폰트는 사용 전 `@font-face`로 등록하세요(파일 상단 주석 참고). 개발 시 값을 직접 입력하지 말고 `var(--color-primary)`처럼 변수로 사용하면 100개 게임의 편차가 사라집니다.

## 3. 화면 사이즈

- **최소 기준** — 아이패드 가로 **1024 × 640** 영역에서 모든 UI가 잘리지 않게 배치합니다.
- **최대 기준** — **전자칠판**(고해상도 대형 화면)에서도 잘 보이도록 구성합니다.
- 모든 그래픽은 **SVG(벡터)**, 레이아웃은 **상대 단위 / 오토레이아웃**으로 구성해 화면 크기에 관계없이 선명하게 합니다.

## 4. 공통 UI 컴포넌트 (자유도 0%)

모든 게임이 동일하게 쓰는 요소입니다. 게임마다 새로 디자인하지 않습니다.

**🔗 Figma 공통 UI가 단일 출처입니다** → ① 공통 UI 링크 (리소스 & 링크 참고)

라이브러리 구성:

- **스플래시스크린** — 게임 로고가 화면 중앙에 표시되는 진입/로딩 화면.
- **버튼**
  - **최소 높이 40px 권장** (터치 기준). 8의 배수로 키웁니다 → `40 / 48 / 56` (`--button-height-*`). 비터치·조밀한 UI에서만 `32px`(`--button-height-sm`) 허용.
  - 가로 폭은 텍스트 길이에 따라 유동(최소사이즈 200px).
- **인풋박스 · 셀렉트박스** — 호버/액티브(포커스) 시 **소마 브랜드 컬러 또는 게임 메인 컬러**를 적용합니다 (`--color-focus`, 게임 메인 컬러로 덮어쓰기 가능).
- **체크박스 / 라디오 버튼** — **UI는 공통 제공**, 컬러는 게임에 맞게 **자유롭게** 사용.
- **게임 소개 화면** — 아래 게임팝업과 동일한 표면 규칙을 따릅니다.
- **게임 팝업**
  - **배경** — 흰색투명도(`--surface-popup-light`) 또는 검은색투명도(`--surface-popup-dark`) + **배경 블러**(`--blur-backdrop`, 뒤 영역에 `backdrop-filter`).
  - **테두리** — **브랜드 컬러** 또는 **게임 메인 컬러**.
  - 별도 모달/얼럿을 새로 만들지 말고 이 팝업 규칙을 재사용하세요.

개발자는 ① 공통 UI 링크에서 직접 export 하거나 스펙을 참조해 구현하세요.

## 6. GUI 아이덴티티 — 두 갈래로 분기

게임의 캐릭터·로고·배경 등 GUI는 **두 가지 방식 중 하나**로 만듭니다.

> ⚠️ **AI 캐릭터와 소마 캐릭터를 혼합해서 사용하지 않습니다.** 한 게임은 한 방식으로 통일합니다.

| 방식 | 자유도 | 문서 |
|---|---|---|
| **AI 생성** | 100% (캐릭터·로고·스플래시·배경 자유) | → **`design_AI.md`** |
| **소마 브랜드** | 활용별 0% / 20% / 50% | → **`design_soma.md`** |

해당 문서에서 캐릭터·게임 로고·게임 배경·결과창 규칙을 확인하세요. (결과창은 점수·정답률 등을 **인포그래픽**으로 시각화하고 브랜드 톤을 유지합니다.)

## 7. 게임 플레이 (자유도 100%)

**공통 컴포넌트(4번)와 화면 흐름(1번)을 제외하면, 인게임 캔버스 내부는 모두 자유입니다.** 재미가 최우선입니다.

- 상호작용(드래그앤드롭 · 터치 · 스와이프), 애니메이션 타이밍, 이펙트, 배경, 게임 요소 — 전부 자유.
- **보상 연출**도 각 게임 고유의 특성에 맡깁니다.
- 자유롭게 만들되 **브랜드 컬러 사용을 권장**합니다.
- 개발자 간 소통 문서를 공유하고, 이 `design.md`를 기준으로 코드를 공통화하세요.

## 부록 A. 개발자 핸드오프 권장 구조

값을 글이 아니라 코드·에셋으로 고정하면 100개 게임의 편차가 사라집니다.

```
/game-shared/
├─ design.md        ← 공통 가이드. AI 코딩 시 컨텍스트로 포함
├─ design_AI.md     ← AI 생성 시 추가
├─ design_soma.md   ← 소마 브랜드 사용 시 추가
├─ tokens.css       ← 컬러·폰트·간격 (모든 게임 import)
└─ svg/             ← 시작용 아이콘 + 기본 도형
   ├─ icons/
   └─ shapes/
```

- 공통 UI·캐릭터·SVG는 **Figma에서 export** (리소스 & 링크 참고).
- 폰트는 `@font-face`로 등록 후 `var(--font-game-*)`로 사용.

## 부록 B. 기획 확정 대기 항목

디자인 규칙이 아니라 **전략 결정**입니다. 정해지면 가이드를 보완합니다. (개발 착수에는 위 내용으로 충분합니다.)

- **소마챔피언즈 정체성** — 기존 소마의 스핀오프(외전)인지, 톤앤매너를 어디까지 맞출지
- **100개의 묶음 성격** — 하나의 세계관/스토리인지, 독립 게임 모음인지 → *통합 플랫폼·뱃지 시스템 필요 여부*
- **게임 그루핑 기준** — 난이도 / 장르 / 협업·경쟁 → *카테고리 컬러 코딩 필요 여부*
- **연동 범위** — 정답가리개, 교재 연동, 디지털 키패드, M-Switch(제로클릭) 연동 범위
