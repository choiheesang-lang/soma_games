# 소마챔피언즈 미니게임 디자인 시스템

> 100개의 미니게임을 **"하나의 제품"으로 보이게** 만들기 위한 공통 디자인 가이드 & 토큰
> v0.3 · 2026.06

## 개요

100개 게임의 "재미"는 살리되 "통일감"은 잃지 않기 위한 최소한의 규칙 모음입니다.
**틀(UI)은 고정 · 아이덴티티는 방식별 부분 자유 · 게임 플레이는 완전 자유.**

## 핵심 원칙

1. **이모지 금지 → SVG 사용** — 해상도(아이패드~전자칠판), 브랜드 컬러 적용, 기기·OS 편차 제거
2. **자유도 3단계**
   | 영역 | 자유도 |
   |---|---|
   | UI 시스템 (공통 컴포넌트·토큰) | 0% (고정) |
   | GUI 아이덴티티 (캐릭터·로고·배경) | 0~50% (방식별) |
   | 게임 플레이 (상호작용·이펙트) | 100% (완전 자유) |

## 브랜드 컬러 (단 AI를 활용할경우 게임 내부에 게임 고유의 컬러를 사용합니다.)

| 토큰 | 값 | 용도 |
|---|---|---|
| `--color-primary` | `#F19A2F` | 주황 — 핵심 강조 |
| `--color-primary-green` | `#69BE00` | 녹색 — 보조 강조 |
| `--color-ink` | `#222824` | 브랜드 검정 |

## 파일 구성

| 파일 | 내용 | 대상 |
|---|---|---|
| `design.md` | 공통 가이드 (토큰·컴포넌트·화면 흐름·SVG·자유도) | 필수 |
| `design_AI.md` | GUI를 **AI로 생성**할 때의 추가 규칙 | AI 생성 담당 |
| `design_soma.md` | GUI를 **소마 브랜드 자산**으로 만들 때의 추가 규칙 | 소마 캐릭터 담당 |
| `tokens.css` | 컬러·폰트·간격·라운드 디자인 토큰 (모든 게임 import) | 개발자 |
| `index.html` | 가이드·토큰·Figma 링크를 한눈에 보는 정리 페이지 | 전원 |

## GUI 두 갈래 (혼합 금지)

- **AI 생성** → `design_AI.md` (자유도 100%)
- **소마 브랜드** (우기·또기·필이) → `design_soma.md` (2D 0% / 3D 20% / 도형 50%)

## 리소스

- **공통 UI (Figma)** — 스플래시·버튼·인풋/셀렉트·게임소개·게임팝업
- **GUI (Figma)** — 캐릭터 이미지 + SVG 아이콘(Phosphor 계열, MIT)
- **디자인 토큰** — `tokens.css`

## 사용법

1. 먼저 `design.md`를 읽습니다.
2. GUI 방식에 따라 `design_AI.md` **또는** `design_soma.md` 중 하나를 추가로 읽습니다.
3. 개발 시 `tokens.css`를 import하고 값은 `var(--token)`으로만 사용합니다.
4. AI 코딩 도구를 쓴다면 저장소에 `.md` 가이드를 컨텍스트로 포함시킵니다.
