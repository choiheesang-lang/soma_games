미니게임 디자인 가이드

100개의 미니게임을
하나의 제품으로 보이게.

"재미"는 살리되 "통일감"은 잃지 않기 위한 최소한의 규칙. 틀(UI)은 고정, 아이덴티티는 방식별 부분 자유, 게임 플레이는 완전 자유 — 이 한 페이지로 전체 가이드를 훑습니다.

5개 파일 보기 Figma 링크
01

전달 파일 5종

공통 가이드 1개 + 방식별 추가 가이드 2개 + 토큰 + 이 정리 페이지. 개발자는 저장소에 .md 파일을 AI 코딩 컨텍스트로 넣고, tokens.css를 import 합니다.

design.md 전원 필수

모든 게임의 공통 메뉴얼. 핵심 원칙(이모지 금지·자유도 3단계), 화면 흐름, 디자인 토큰, 공통 UI 컴포넌트, SVG 사용 규칙, 게임 플레이 자유도. 나머지 두 문서는 이 문서를 보충합니다.

design_AI.md AI 생성

GUI를 AI로 생성할 때의 추가 규칙. 자유도 100%, 브랜드 컬러 자유 사용, 상태값 5색은 고정.

design_soma.md 소마 브랜드

GUI를 소마 브랜드 자산으로 만들 때의 추가 규칙. 활용별 자유도 0% / 20% / 50%.

tokens.css 코드

컬러·폰트·간격·라운드·버튼·팝업의 단일 출처. 모든 게임이 import 해 var(--token)으로 사용.

index.html 개요

지금 보고 있는 이 페이지. 다섯 파일과 Figma 링크, 토큰을 한눈에 정리한 허브.

02

핵심 원칙

두 가지만 이해하면 나머지는 참조용입니다.

이모지 금지 → SVG

모든 아이콘·기호·장식은 이모지가 아니라 SVG로 만듭니다. 해상도(아이패드~전자칠판), 브랜드 컬러 적용, 기기·OS 편차 제거를 위해서입니다.

✕ 🎮 🔊 ✅ → ○ SVG (Phosphor)
영역자유도내용
UI 시스템0%공통 컴포넌트 · 디자인 토큰 — 100개 전부 동일
GUI 아이덴티티0~50%캐릭터 · 로고 · 배경 — 방식별(AI / 소마)
게임 플레이100%인게임 상호작용 · 이펙트 — 완전 자유

06

GUI는 두 갈래

캐릭터·로고·배경은 두 방식 중 하나로 만듭니다. 해당 문서를 추가로 읽으세요.

⚠️ AI 캐릭터와 소마 캐릭터를 한 게임에서 혼합하지 않습니다.
design_AI.md
AI 생성
자유도 100%
  • 캐릭터 2D/3D 자유 생성 (브랜드 컬러 반영 권장)
  • 게임 로고 · 스플래시 · 배경 게임에 맞게 자유
  • 브랜드 컬러 자유롭게 사용

design_soma.md
소마 브랜드
자유도 0% / 20% / 50%
  • 2D 활용 (0%) 우기·또기·필이 변형 없이 그대로
  • 3D 활용 (20%) AI 3D 변환, 형태 유지 (GPT 권장)
  • 도형 활용 (50%) 삼각형·사각형·원으로 캐릭터 생성
  • 로고·배경 녹색·주황·블랙 조화
